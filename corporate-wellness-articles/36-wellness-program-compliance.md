## Introduction

Biometric wellness programs handle sensitive data—health information that's protected by stringent regulations. GDPR in Europe, HIPAA in the US, and various country-specific laws create a complex compliance landscape. Understanding requirements is essential before deployment.

---

## GDPR Requirements

### Core Principles

| Principle | Application |
|-----------|-------------|
| Lawful basis | Consent or legitimate interest |
| Purpose limitation | Defined, specific use |
| Data minimization | Only what's needed |
| Accuracy | Keep data correct |
| Storage limitation | Time-limited retention |
| Security | Appropriate protections |

### Employee Rights

| Right | Implementation |
|-------|----------------|
| Access | View their data |
| Rectification | Correct errors |
| Erasure | Delete on request |
| Portability | Export their data |
| Object | Opt out |
| Withdraw consent | Stop processing |

### Consent Requirements

| Element | Requirement |
|---------|-------------|
| Freely given | No pressure or penalty |
| Specific | Clear purpose |
| Informed | Understand processing |
| Unambiguous | Clear affirmation |
| Withdrawable | Easy to revoke |

## HIPAA Considerations

### When HIPAA Applies

| Scenario | HIPAA Status |
|----------|--------------|
| Employer-sponsored wellness | May apply |
| Integrated with health plan | Likely applies |
| Standalone wellness | May not apply |
| Vendor handling PHI | BAA required |

### Key Requirements

| Requirement | Purpose |
|-------------|---------|
| Privacy Rule | Use and disclosure limits |
| Security Rule | Technical safeguards |
| BAA | Vendor agreements |
| Breach notification | Incident response |

### Practical Implications

| Consideration | Action |
|---------------|--------|
| Data segregation | Separate wellness data |
| Minimum necessary | Limit access |
| Authorization | Employee consent |
| Training | Staff awareness |

## Program Design for Compliance

### Voluntary Participation

| Requirement | Implementation |
|-------------|----------------|
| No coercion | Truly optional |
| No adverse consequences | Equal treatment |
| Easy withdrawal | Simple opt-out |
| Incentive limits | Reasonable amounts |

### Data Protection

| Protection | Implementation |
|------------|----------------|
| Encryption | At rest and in transit |
| Access controls | Role-based, minimal |
| Audit trails | Complete logging |
| Retention limits | Defined periods |

### Transparency

| Element | Communication |
|---------|---------------|
| Privacy notice | What's collected, why |
| Data use | How it's processed |
| Data sharing | Who sees what |
| Rights | How to exercise |
