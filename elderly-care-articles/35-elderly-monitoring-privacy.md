## Introduction

Privacy is a legitimate concern for seniors and families considering health monitoring. Understanding exactly what data is collected, who can see it, and how it's protected helps make informed decisions and have honest conversations.

---

## What Data Is Collected

### Typical Data Points

| Data Type | What's Captured |
|-----------|-----------------|
| Heart rate | Continuous heart rate readings |
| HRV | Heart rate variability metrics |
| Activity | Steps, movement, active minutes |
| Sleep | Duration, quality, stages |
| Device data | Battery, connectivity |

### What's NOT Typically Collected

| Not Collected | Clarification |
|---------------|---------------|
| Location | Unless specific feature enabled |
| Audio | Not listening |
| Video | No cameras |
| Phone calls | Not recorded |
| Messages | Not accessed |
| Browsing | Not tracked |

---

## Who Can See Data

### Access Levels

| Person | What They See |
|--------|---------------|
| Senior (patient) | All their own data |
| Authorized family | Summary, alerts, trends |
| Healthcare provider | If shared by patient |
| Platform company | Aggregated, anonymized |

### Family Access

| Feature | Visibility |
|---------|------------|
| Daily summaries | Overall status |
| Alert notifications | When thresholds crossed |
| Trend views | Patterns over time |
| Detailed data | Configurable |

### What Family Cannot See

| Privacy Protection | Description |
|-------------------|-------------|
| Location details | Not tracked |
| Minute-by-minute | Summary, not constant |
| Personal communications | Never accessed |
| Lifestyle details | Beyond health metrics |

## Data Protection

### Security Measures

| Protection | Description |
|------------|-------------|
| Encryption | Data encrypted in transit and at rest |
| Secure login | Password/biometric protection |
| Access controls | Role-based visibility |
| Audit logs | Who accessed what |

### Regulatory Compliance

| Regulation | Protection |
|------------|------------|
| HIPAA | US health data protection |
| GDPR | European privacy law |
| State laws | Additional protections |
| Industry standards | Security certifications |

## Who Else Might Access Data

### Third-Party Access

| Entity | Access Level |
|--------|--------------|
| Platform company | Aggregate, anonymized for improvement |
| Cloud providers | Infrastructure only, encrypted |
| Healthcare integrations | Only with explicit consent |
| Researchers | Anonymized, opt-in only |

### Who Does NOT Get Data

| Entity | Explicitly Excluded |
|--------|---------------------|
| Insurance companies | Not shared |
| Employers | Not applicable |
| Advertisers | Not sold |
| Government | Without legal process |

## Senior Control

### Their Rights

| Right | Implementation |
|-------|----------------|
| Access | See all their data |
| Correction | Fix inaccurate data |
| Deletion | Request data removal |
| Portability | Export their data |
| Revoke sharing | Stop family access |

### Control Options

| Control | Description |
|---------|-------------|
| Who gets access | Add/remove family |
| Alert settings | What triggers notifications |
| Data sharing | With healthcare providers |
| Account deletion | End participation |

---

## Addressing Concerns

### Common Questions

| Question | Answer |
|----------|--------|
| Is it always tracking? | Health data only, not surveillance |
| Who owns the data? | Senior owns their data |
| Can it be hacked? | Robust security, but no system perfect |
| Will it be sold? | Not sold to third parties |
| Can I see what's collected? | Full transparency available |

### Having the Conversation

| Approach | Effect |
|----------|--------|
| Be transparent | Explain exactly what's collected |
| Show controls | Demonstrate their options |
| Respect concerns | Take privacy seriously |
| Document limits | What's NOT collected |

---

## Key Takeaways

1. **Know what's collected:** Health data only
2. **Location usually not tracked:** Unless specific feature
3. **Access is controlled:** Senior + authorized family
4. **Data is protected:** Encryption, security measures
5. **Compliance required:** HIPAA, GDPR as applicable
6. **Not sold:** To advertisers or third parties
7. **Senior controls:** They can revoke access
8. **Transparency:** Can see what's collected
